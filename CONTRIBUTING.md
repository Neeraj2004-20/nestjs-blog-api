# Contributing to Blog API

Thank you for your interest in contributing! Here's how you can help.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR-USERNAME/blog-api.git`
3. Create a new branch: `git checkout -b feature/your-feature-name`
4. Install dependencies: `npm install`
5. Start development: `npm run start:dev`

## Development Workflow

### Making Changes

1. Make your changes in a new branch
2. Follow the existing code style
3. Write tests for new features
4. Test locally: `npm run start:dev`

### Commit Messages

Use clear, descriptive commit messages:
- `feat: Add user authentication`
- `fix: Resolve post creation issue`
- `docs: Update API documentation`
- `test: Add tests for auth service`

### Code Style

- Use TypeScript for all code
- Follow NestJS conventions
- Use meaningful variable names
- Add JSDoc comments for complex functions

## Submitting Changes

1. Push your branch to your fork
2. Create a Pull Request with a clear description
3. Include any relevant issue numbers
4. Wait for review and address feedback

## Reporting Bugs

1. Check if the bug is already reported
2. Create an issue with:
   - Clear title and description
   - Steps to reproduce
   - Expected vs actual behavior
   - Your environment details

## Feature Requests

1. Check existing issues/discussions
2. Provide clear use case
3. Suggest implementation approach if possible

## Code of Conduct

- Be respectful and inclusive
- Welcome diverse perspectives
- Focus on the code, not the person
- Help others learn and grow

## Questions?

Feel free to open a discussion or issue for questions!
